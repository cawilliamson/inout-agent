## crate

this is a first-party extension crate. it depends on `inout-core` and implements `inout_core::Extension` to register observability hooks and the subscriber bus. the binary crate `crates/inout` loads it by default.

```rust

    fn version(&self) -> &'static str { "0.3.0" }
    fn register(&self, api: &mut ExtensionApi) {
        // see sections below
    }
}
```
## scope

make the agent observable without binding core to any vendor (otel, sentry, datadog). emit stable structured lifecycle events. capture the raw provider payload â€” the "raw stream" â€” so you can see exactly what prompts are exchanged.

### in scope

1. `Observability` trait + default in-memory subscriber bus.
2. trace/span event types (`start`/`end`/`error`/`event`).
3. raw payload capture via `before_provider_payload` hook + `after_provider_response`.
4. `~/.inout/llm.log` â€” request/response direction log.
5. `~/.inout/llm_requests/<ts>_<slug>.json` â€” per-request full payload dump.
6. `~/.inout/audit.jsonl` â€” every tool call, every permission decision.
7. per-turn cost display (tokens in/out/cache, estimated usd).

### not in scope

- otel/sentry adapters (third-party crates, not core).
- metrics aggregation (out of scope; events are the substrate).

## design rationale (mined from pi + zap)

### from pi: `PiObservability` abstraction

pi's `packages/agent/docs/observability.md` defines the exact model we want:

> pi owns a small runtime-agnostic observability abstraction. external listeners convert those events into otel spans, sentry spans, logs, metrics.

pi's `traceOperation`:
1. reads current context
2. creates `traceId` if missing
3. creates `spanId`
4. uses current span as `parentSpanId`
5. emits `start`
6. runs callback under child context
7. emits `end` or `error`

event names: `pi.agent.prompt`, `pi.agent.turn`, `pi.agent.tool_call`, `pi.ai.provider.request`, `pi.session.append_entry`, etc. each emits `start`/`end`/`error`.

pi's safety defaults are right: payloads are unsafe by default (prompts, completions, tool args, tool results, shell output, file contents, provider request/response, api keys, headers). safe by default: provider, model, session id, entry type, tool name, status code, stop reason, token counts, costs, durations.

we take this verbatim, translated to rust. the `tracing` crate is the obvious substrate but pi deliberately avoids binding to it for the *public* event surface â€” the bus is a thin trait and listeners adapt to `tracing`/otel/sentry. we follow.

### from zap: raw stream files

zap's `src/log.rs` writes:
- `~/.zap/llm.log` â€” one direction block per request/response, pretty-printed json.
- `~/.zap/llm_requests/<ts>_<slug>.json` â€” full non-stream request body, saved so you can `curl` it back.
- `~/.zap/audit.jsonl` â€” one json record per tool call.
- rotation on startup: trim to last 24h.

zap's `src/session/turn.rs` computes per-turn cost:
```
total_usd = (input_tokens * cost_in + output_tokens * cost_out) / 1_000_000
```
plus `cache_read_tokens`, `cache_write_tokens`. the breakdown shows: skills tokens, msg tokens, ctx pct bar, cost string.

### what we keep from each

| idea | source | inout shape |
|---|---|---|
| typed event bus, subscriber model | pi | `Observability` trait + `EventBus` |
| `traceOperation` span emission | pi | `trace_span()` helper |
| raw payload dump files | zap | `~/.inout/llm_requests/` |
| direction log | zap | `~/.inout/llm.log` |
| audit jsonl | zap | `~/.inout/audit.jsonl` |
| per-turn cost breakdown | zap | `TurnCost` struct, tui/cli display |
| safe-by-default payload redaction | pi | `RedactionPolicy` enum |

## components

### observability trait

file: `crates/inout-ext-observability/src/mod.rs`

```rust
pub struct SpanContext {
    pub trace_id: String,
    pub current_span_id: Option<String>,
    pub user_context: Option<Arc<Map<String, Value>>>,
}

pub enum EventKind { Start, End, Error, Event }

pub struct ObservabilityEvent {
    pub kind: EventKind,
    pub name: &'static str,
    pub trace_id: String,
    pub span_id: String,
    pub parent_span_id: Option<String>,
    pub timestamp: u64, // ms since epoch
    pub duration_ms: Option<u64>,
    pub payload: Redacted<Value>,
    pub error: Option<SerializedError>,
}

pub trait Observability: Send + Sync {
    fn current_context(&self) -> Option<SpanContext>;
    fn run_with_context<R>(&self, ctx: SpanContext, f: impl FnOnce() -> R + Send) -> R where R: Send;
    fn emit(&self, event: ObservabilityEvent);
    fn has_subscribers(&self) -> bool;
}

pub fn trace_span<R>(obs: &dyn Observability, name: &'static str, payload: Redacted<Value>, f: impl FnOnce() -> R + Send) -> R where R: Send;
```

### redaction

```rust
pub enum RedactionPolicy { Safe, UnsafeOptIn }

pub struct Redacted<T> {
    pub value: Option<T>,
    pub policy: RedactionPolicy,
}
```

default policy is `UnsafeOptIn` with `value: None` for sensitive fields. content capture requires explicit `INOUT_OBSERVE_CONTENT=1`.

### raw stream capture

file: `crates/inout-ext-observability/src/raw_stream.rs`

hook into `before_provider_payload` and `after_provider_response`:

```rust
fn install_raw_stream_logger(hooks: &mut HookBus, sink: RawStreamSink) {
    hooks.on::<BeforeProviderPayloadEvent, _, _>(move |ev, _ctx| {
        let slug = slugify(&ev.model);
        let ts = unix_ms();
        let path = sink.requests_dir.join(format!("{ts}_{slug}.json"));
        std::fs::write(&path, serde_json::to_vec_pretty(&ev.payload).unwrap_or_default()).ok();
        sink.append_llm_log(&format!("REQUEST [{}] -> {}", ev.model, path.display()));
        BeforeProviderPayloadResult { payload: ev.payload.clone() }
    });

    hooks.on::<AfterProviderResponseEvent, (), _>(move |ev, _ctx| {
        sink.append_llm_log(&format!("RESPONSE [{}] status={}", ev.model, ev.status));
    });
}
```

### audit log

file: `crates/inout-ext-observability/src/audit.rs`

zap's audit is minimal: `{ timestamp, event }`. we extend with structured fields:

```rust
#[derive(Serialize)]
pub struct AuditRecord {
    pub ts: String,           // rfc3339
    pub event: &'static str,  // "tool_call" | "permission_decision" | "user_turn" | "llm_response"
    pub tool: Option<&'static str>,
    pub permission: Option<PermissionDecision>,
    pub affected_path: Option<PathBuf>,
    pub exit_code: Option<i32>,
    pub duration_ms: Option<u64>,
    pub trace_id: Option<String>,
    pub span_id: Option<String>,
}
```

appended to `~/.inout/audit.jsonl`. buffered writer, flushed per-record. rotated on startup to last 7d.

pub struct TurnCost {

## acceptance

- [ ] `inout-core` builds and passes clippy without `inout-ext-observability`.
- [ ] `trace_span` emits `start`+`end` (or `error`) around any closure.
- [ ] `before_provider_payload` hook writes `~/.inout/llm_requests/<ts>_<slug>.json` with the full payload.
- [ ] `~/.inout/llm.log` gets one `REQUEST` and one `RESPONSE` line per provider call.
- [ ] `~/.inout/audit.jsonl` gets one record per tool call, one per permission decision.
- [ ] `Redacted<T>` with default policy never serialises prompt/completion content.
- [ ] `INOUT_OBSERVE_CONTENT=1` enables content capture.
- [ ] logs rotate on startup (llm.log to 24h, llm_requests to 24h, audit to 7d).
- [ ] per-turn `TurnCost` is computable from `Usage` + skill trace + context window estimate.
- [ ] `cargo clippy --all-targets -- -D warnings` green.
