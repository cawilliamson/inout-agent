## crate

this is a first-party extension crate. it depends on `inout-core` and implements `inout_core::Extension` to register skill hooks and `/skill` commands. the binary crate `crates/inout` loads it by default.

## scope

markdown skill files with yaml frontmatter. lazy-loaded, trigger-matched per turn. injected into the system prompt only when relevant. this is zap's core bet against prompt bloat, and it is the right bet.

### in scope

1. `Skill` struct + frontmatter parsing.
2. `SkillLoader` ‚Äî discovers skills from `~/.inout/skills/`, `.inout/skills/`, config `skill_paths`, and compiled-in defaults.
3. `TriggerEngine` ‚Äî keyword match with word-boundary rules.
4. `SkillBudget` ‚Äî rank, truncate, always-on block under a token cap.
5. `SkillTrace` ‚Äî per-turn record of which skills fired and why.
6. stack auto-detection from manifest files.
7. `/skill` commands (list, show, create, export, scope, log).

### not in scope

- semantic/vector skill routing (keyword trigger is enough).
- cross-agent skill install from github (v3 idea).

## design rationale (mined from zap)

zap's `src/skill_manager.rs` is the model. three things right:

1. **three categories.** `Core` (always-on, baked into base system prompt), `Practice` (always a trigger candidate across stacks ‚Äî git, debugging, code-review), `Domain` (session-scoped; only trigger-matchable after startup). avoids the "always inject everything" trap.

2. **trigger word matching with length-aware boundaries.** zap's `trigger_word_match` (lines 26-60):
   - non-alphabetic triggers (`.rs`, `#include`, `fn `): simple substring ‚Äî inherently anchor-like.
   - short alphabetic (<=3 chars, e.g. `pr`, `go`, `sql`): require BOTH word boundaries, so `pr` does not match `process`.
   - longer alphabetic (>=4 chars, e.g. `review`): require only word-START boundary, so `reviewing` matches `review` but `preview` does not.

   we take it verbatim.

3. **token budget with ranking.** `rank_and_truncate_skills` sorts by priority (desc) -> source tier (project > external > global > bundled) -> token count (asc, prefer smaller). pinned skills always kept. budget exists because always-on skills are paid on EVERY prompt of every session ‚Äî unbounded block silently taxes cost and makes local models unusable.

what zap does that we keep:
- `SkillCategory` enum.
- `SkillSource` tiers: `Bundled`, `Global`, `External` (from `skill_paths`), `Project`.
- `load_all_skills` ‚Äî bundled first, then global, then extra, then project; higher-priority same-name overrides.
- `build_always_on_prompt_budgeted` ‚Äî returns the block, the included token total, and names of skills dropped by budget.
- `assemble_always_on_block` ‚Äî block + optional user-facing warning when skills dropped.
- stack auto-detection: `STACK_MANIFESTS` maps `Cargo.toml` -> `["rust"]`, `package.json` -> `["typescript", "react"]`, etc. `detect_domain_scope` populates the session's domain scope at startup.
- `ensure_triggers` ‚Äî foreign skills (claude code `SKILL.md` without `triggers:`) get name-derived triggers so they still activate on-demand.

what we drop from zap:
- `bootstrap_bundled_skills` writing defaults to disk on first run. inout keeps bundled skills compiled in; user overrides via `~/.inout/skills/` only. less filesystem pollution.
- the `print_repl_skill_summary` terminal formatting (lives in tui layer, not core).

## components

### skill struct

file: `crates/inout-ext-skills/src/mod.rs`

```rust
#[derive(Debug, Clone, PartialEq)]
pub enum SkillCategory { Core, Practice, Domain }

#[derive(Debug, Clone, PartialEq)]
pub enum SkillSource { Bundled, Global, External, Project }

#[derive(Debug, Clone)]
pub struct Skill {
    pub name: String,
    pub description: String,
    pub category: SkillCategory,
    pub source: SkillSource,
    pub triggers: Vec<String>,
    pub priority: i32,
    pub token_estimate: usize,
    pub content: String,
    pub file_path: PathBuf,
}

impl Skill {
    pub fn is_always_on(&self) -> bool { self.category == SkillCategory::Core }
    pub fn tokens(&self) -> usize {
        if self.token_estimate > 0 { self.token_estimate } else { self.content.len() / 4 }
    }
}
```

### frontmatter

```yaml
---
name: rust
category: domain
trigger: ["rust", "cargo", "fn ", "trait "]
priority: 10
tokens: 700
---
body content here
```

parsing via `serde_yaml` for the frontmatter block between `---` fences. unknown fields ignored (foreign skill compat). missing `trigger` -> `Core` category. missing `category` -> `Practice` for foreign skills with a description, `Domain` otherwise.

### loader

file: `crates/inout-ext-skills/src/loader.rs`

```rust
pub fn skill_dirs(extra: &[String]) -> Vec<(PathBuf, SkillSource)>;
pub fn load_all_skills(extra_dirs: &[String]) -> Vec<Skill>;
```

priority order (lowest -> highest, later wins on name collision):
1. `bundled_skills()` ‚Äî compiled in via `include_str!`.
2. `~/.inout/skills/` ‚Äî global.
3. `skill_paths` entries left-to-right ‚Äî external (kiro, claude, custom).
4. `.inout/skills/` ‚Äî project.

### skill trace

file: `crates/inout-ext-skills/src/trace.rs`

```rust
#[derive(Debug, Clone, Serialize)]
pub struct SkillTraceEntry {
    pub turn: usize,
    pub user_preview: String,    // first 60 chars
    pub matched_skills: Vec<String>,
    pub reason: Option<String>,  // "casual" | "no match" | None
}

pub struct SkillTrace { entries: Vec<SkillTraceEntry> }

impl SkillTrace {
    pub fn push(&mut self, turn: usize, preview: &str, skills: &[&Skill], reason: Option<&str>);
    pub fn for_turn(&self, turn: usize) -> Option<&SkillTraceEntry>;
    pub fn all(&self) -> &[SkillTraceEntry];
}
```

this is zap's `skill_trace` field on Session ‚Äî visible via `/skill log`. audit-friendly.

### commands

`/skill list` ‚Äî grouped: always-on / triggered, with source glyph.
`/skill show <name>` ‚Äî preview content + description.
`/skill create <name>` ‚Äî scaffold in `.inout/skills/`.
`/skill create <name> --global` ‚Äî scaffold in `~/.inout/skills/`.
`/skill log` ‚Äî skill trace for this session.
`/skill scope` ‚Äî show/change active domain skills.
