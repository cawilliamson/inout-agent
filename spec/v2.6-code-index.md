## scope

an optional sqlite-backed ast symbol index, built via tree-sitter. behind a feature flag, default-off. the core loop must build and run with this disabled.

### in scope

1. `CodeIndex` — sqlite at `.inout/code.db`, opened in-memory for tests.
2. `Symbol`, `CallSite`, `Import`, `TypeEdge` types.
3. tree-sitter extractors per language: rust, python, typescript, javascript, go, java.
4. incremental reindex on file write.
5. background indexer (env-configurable interval, default 120s).
6. powered tools: `code_map`, `find_definition`, `find_references`, `who_calls`, `file_imports`, `where_imported`, `find_subtypes`/`find_supertypes`, `pack_context`, `ripple_analysis`, `get_diagnostics`, `lsp_definition`, `lsp_type_at`.
7. `/index` commands: reindex, stats, quality.
8. index usage logging (hit/miss per turn, grep fallback).

### not in scope

- vector/embedding search (deliberately omitted — ast is structural, not semantic).
- cross-repo indexing (single repo only).

## design rationale (mined from zap)

zap `src/code_index/` is the model. the case for indexing vs pure agentic search:

> every time claude code answers "where is `UserRepository` defined?", the model spends tokens deciding which files to read, issues multiple grep calls, reads partial file content, synthesises the answer. the index answers the same question in a single sqlite lookup. on a 50k-line repo, that is the difference between 1 tool call and 5.

the index owns **structural** questions. grep owns **semantic** ones. every `find_definition` miss falls back to ripgrep. so the agent never does worse than pure agentic search, only better when a symbol is indexed.

what zap does that we keep:
- `Symbol` struct: `{ path, line, kind, name, signature? }`.
- `CallSite`: `{ call_path, line, callee_name, qualifier? }`.
- `Import`: `{ path, imported_name, kind, line }`.
- `TypeEdge`: `{ child_name, parent_name, kind: "implements" | "extends" }`.
- `PackedItem` / `PackedContext` — auto-select most load-bearing files for a task within a token budget. used by `pack_context`.
- `QualityReport` — god objects (>15 methods), high coupling, dead code candidates, quality score (0-100).
- `global_*` free functions over a `OnceLock<Arc<Mutex<CodeIndex>>>` singleton.
- `spawn_background_indexer` — reindexes changed files every 120s.
- `run_index_standalone` — `inout --index-only` builds the index with no session, no LLM call.
- per-tool-call logging: `[INDEX] hit · find_definition · 'UserRepository' · 3 result(s)` / `[INDEX] miss · find_definition · 'legacy_fn' · grep fallback`.
- `peek_scan_progress` — open a read-only sqlite connection during a scan to report live progress (wal mode).

what we drop from zap:
- nothing material. the index is the right design.

### symbol

```rust
#[derive(Debug, Clone)]
pub struct Symbol {
    pub path: String,
    pub line: usize,
    pub kind: String,   // "function" | "struct" | "trait" | "class" | "method" | "enum"
    pub name: String,
    pub signature: Option<String>,
}

#[derive(Debug, Clone)]
pub struct CallSite {
    pub call_path: String,
    pub line: usize,
    pub callee_name: String,
    pub qualifier: Option<String>,
}

#[derive(Debug, Clone)]
pub struct Import {
    pub path: String,
    pub imported_name: String,
    pub kind: String,
    pub line: usize,
}

#[derive(Debug, Clone)]
pub struct TypeEdge {
    pub child_name: String,
    pub parent_name: String,
    pub kind: String,  // "implements" | "extends"
}
```

### extractors

file: `crates/inout-ext-ast/src/extract_*.rs`

one per language. each walks the tree-sitter ast and emits `Symbol`, `CallSite`, `Import`, `TypeEdge` rows. languages: rust, python, typescript, javascript, go, java. `tree-sitter-c-sharp` optional.

### quality report

```rust
#[derive(Debug, Default)]
pub struct QualityReport {
    pub score: u32,         // 0-100
    pub god_objects: Vec<(String, String, usize)>,  // (name, path, method_count)
    pub high_coupling: Vec<(String, usize)>,         // (symbol, ref_count)
    pub dead_code: Vec<(String, String)>,            // (name, path)
}
```

`/index quality` renders this in the tui.
