## crate

this is a first-party extension crate. it depends on `inout-core` and provides http llm providers. the binary crate `crates/inout` loads it by default. `ReplayLlmClient` stays in `inout-core` for core tests.

## scope

the provider trait, streaming, prompt caching, retry, and the two v1.0 clients (replay + http). this extends v1.0 `LlmClient` trait with the observability hooks + raw-stream capture + retry/caching.

### in scope

1. `LlmProvider` trait — streaming-first, send + sync.
2. `ContentBlock` enum — text, tool_use, tool_result, image.
3. `Message` struct — role + content blocks.
4. `Usage` — input/output/cache_read/cache_write tokens.
5. `ApiResponse` — content + usage + stop_reason.
6. `BeforeOutput` callback — fires on first stream chunk (stops spinner).
7. `ReplayLlmClient` — v1.0, returns pre-recorded responses.
8. `HttpLlmClient` — anthropic + openai-compatible, wired in v2.
9. `send_with_retry` — exponential backoff on stream drop.
10. anthropic prompt caching — `cache_control` breakpoints on system prompt + tools + final message.
11. per-task model routing — `model_routes` config for coding/review/explain/search.

### not in scope

- local slm execution watchdog (v3).
- bedrock/vertex (v3).

## design rationale (mined from zap)

zap `src/llm_client/mod.rs` + `src/llm_client/anthropic.rs` + `src/llm_client/openai.rs` are the model.

key types from zap:

```rust
#[derive(Debug, Clone, Serialize, Deserialize)]
#[serde(tag = "type", rename_all = "snake_case")]
pub enum ContentBlock {
    Text { text: String },
    ToolUse { id: String, name: String, input: serde_json::Value },
    ToolResult { tool_use_id: String, content: String },
    Image { media_type: String, data: Vec<u8> },
}

#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct Message { pub role: String, pub content: Vec<ContentBlock> }

#[derive(Debug, Clone, Default)]
pub struct Usage {
    pub input_tokens: u64,
    pub output_tokens: u64,
    pub cache_read_tokens: u64,
    pub cache_write_tokens: u64,
}

#[derive(Debug)]
pub struct ApiResponse {
    pub content: Vec<ContentBlock>,
    pub usage: Option<Usage>,
    pub stop_reason: Option<String>,
}

pub type BeforeOutput = Box<dyn FnOnce() + Send>;

#[async_trait]
pub trait LlmProvider: Send + Sync {
    async fn send(
        &self,
        system: &str,
        messages: &[Message],
        tools: &[serde_json::Value],
        before_output: Option<BeforeOutput>,
        thinking_budget: Option<u32>,
    ) -> Result<ApiResponse>;
}
```

what zap does that we keep:
- `BeforeOutput` callback fires on first stream chunk — stops the spinner before the first token.
- `send_with_retry` — exponential backoff on stream drop (`sse stream error`, `connection reset`, `broken pipe`), up to 4 retries with 3s/6s/12s/24s delays.
- `check_tool_support_error` — detect provider-specific "tool use not supported" errors and surface clearly.
- `check_text_mode_tool_call` — detect models that chat code instead of calling tools (SLM loop detection).
- `build_curl_block` — for the raw-stream log, generate a `curl` command from the request so you can replay it.
- anthropic prompt caching: `cache_control` breakpoints on system prompt + tool definitions, plus the final message for reuse on the next request.
- `redact_token` — never log full api keys.
- per-task model routing via `model_routes` config: classify prompt as `coding`/`review`/`explain`/`search`, swap model for that turn, restore after.
- `provider_supports_vision` + `local_model_supports_vision` — heuristics for image content.
- `create_client` factory — returns `Box<dyn LlmProvider>` based on config.

what we tighten:
- `tracing` spans around every provider call (via v2.1 `trace_span`).
- raw payload captured via the `before_provider_payload` hook (v2.1), not a bespoke logger.
