## crate

security is split into first-party extension crates: `inout-ext-permissions`, `inout-ext-secret-scanner`, `inout-ext-jail`, `inout-ext-undo`. all depend on `inout-core` and implement `inout_core::Extension` to register hooks. the binary crate `crates/inout` loads them by default.

each subcrate implements `Extension`:

```rust
// example: crates/inout-ext-permissions/src/lib.rs
use inout_core::{Extension, ExtensionApi};

#[derive(Default)]
pub struct PermissionsExtension;

impl Extension for PermissionsExtension {
    fn name(&self) -> &'static str { "permissions" }
    fn version(&self) -> &'static str { "0.3.0" }
    fn register(&self, api: &mut ExtensionApi) {
        // see sections below
    }
}
```

## scope

the agent handles source code, credentials, and shell. security is a core feature. four layers: permission modes, secret scanner, shell sandbox, audit trail.

### in scope

1. `PermissionMode` ŌĆö `Ask` (default), `Auto`, `Deny`.
2. `PermissionManager` ŌĆö quick check, session grants, grouped batch prompt.
3. `SecretScanner` ŌĆö 25+ patterns, block-before-send, entropy heuristic.
4. `ShellSandbox` ŌĆö `Off`, `Workdir`, `Container` modes.
5. `AuditTrail` ŌĆö every tool call + every permission decision to `~/.inout/audit.jsonl`.
6. `UndoLedger` ŌĆö in-memory snapshot before every edit, `/undo` restores.

### not in scope

- network firewalling (out of core scope).
- container image policy (operator concern).

## design rationale (mined from zap)

zap security stack is the best of any agent reviewed. four files do the work:

1. **`src/permission_manager.rs`** ŌĆö three modes. `Ask` prompts for write tools (`shell`, `write_file`, `edit_file`, `batch_edit`, `spawn_agent`). `Auto` approves all. `Deny` blocks all writes. key idea: **tool grant classes** ŌĆö `tool_grant_class` groups `edit_file`/`write_file`/`batch_edit` so granting one grants all semantically-identical ops. `/init` pre-grants via `grant_session` because the user consented by invoking it. **grouped batch prompt** ŌĆö `prompt_batch` shows ONE prompt for all calls needing input in a turn, returns parallel bool vec. avoids prompt fatigue.

2. **`crates/inout-ext-secret-scanner/src/lib.rs`** ŌĆö 25+ patterns: anthropic (`sk-ant-`), openai (`sk-proj-`, `sk-or-`), stripe, google (`aiza`), hugging face (`hf_`), github (`ghp_`/`ghs_`/`gho_`/`ghu_`/`ghr_`/`github_pat_`), gitlab (`glpat-`), npm (`_authtoken=`), aws (`akia`, `aws_secret_access_key`), gcp service account json, azure, slack (`xoxb-`/`xoxp-`/`xapp-`), db connection strings (`postgres://`, `mysql://`, `mongodb://`, `redis://`), private key blocks (`-----begin`), jwt (`eyjh`/`eyja`), generic fields (`password=`, `api_key=`, `secret=`, `access_token=`, `authorization: bearer `). plus `line_has_high_entropy_secret` ŌĆö shannon entropy + mixed character classes, tuned to avoid false positives on git SHAs (all-lowercase hex). `redact` strips secret lines in-place, returns summary for user notice.

3. **shell sandbox** (in `zap/src/tools/shell.rs`) ŌĆö `SandboxMode::Off` / `Workdir` (sets `current_dir` to project root) / `Container` (wraps via docker/podman with `--network none` and read-only mounts).

4. **`src/audit.rs`** ŌĆö minimal `{ timestamp, event }` jsonl. we extend this in v2.1 with structured fields.

what we keep: all of it. what we tighten: audit record structure (v2.1), and we add the `/undo` ledger (zap has it in `zap/src/tools/undo.rs` ŌĆö snapshot before edit, restore on `/undo <path>`).

## components

### permission manager

file: `crates/inout-ext-permissions/src/permission.rs`

```rust
#[derive(Debug, Clone, Copy, PartialEq, Eq)]
pub enum PermissionMode { Ask, Auto, Deny }

pub enum QuickDecision { Allow, Deny, NeedsPrompt }

pub struct PermissionManager {
    pub mode: PermissionMode,
    session_grants: HashMap<String, bool>,
}

impl PermissionManager {
    pub fn new(mode: PermissionMode) -> Self;
    pub fn grant_session(&mut self, tools: &[&str]);
    pub fn is_session_granted(&self, tool_name: &str) -> bool;
    pub fn quick_check(&self, tool_name: &str) -> QuickDecision;
    pub async fn prompt_batch(&mut self, pending: &[(String, String, String)]) -> Result<Vec<bool>>;
}
```

`WRITE_TOOLS` = `["shell", "write_file", "edit_file", "batch_edit", "spawn_agent"]`. `tool_grant_class` maps a tool to its grant group. `prompt_batch` shows one grouped prompt per turn.

### secret scanner

file: `crates/inout-ext-secret-scanner/src/lib.rs`

```rust
#[derive(Debug)]
pub struct SecretMatch {
    pub pattern_name: &'static str,
    pub line: usize,
    pub preview: String,  // first 30 chars, rest ***
}

pub fn scan(content: &str) -> Vec<SecretMatch>;
pub fn redact(content: &mut String, hits: &[SecretMatch]) -> String;
fn line_has_high_entropy_secret(line: &str) -> bool;
fn shannon_entropy_bits_per_char(s: &str) -> f64;
```

pattern table mirrors zap 25+ entries (see rationale). entropy threshold: long + high entropy + mixed character classes, excluding all-lowercase-hex (git SHAs).

### shell sandbox

file: `crates/inout-ext-permissions/src/sandbox.rs`

```rust
#[derive(Debug, Clone, Copy, PartialEq, Eq)]
pub enum SandboxMode { Off, Workdir, Container }

pub fn build_command(argv: &[String], mode: SandboxMode, project_root: &Path) -> Result<Command>;
```

`Container` wraps via docker/podman with `--network none`, read-only mounts of project root, workdir set to project root. detected at startup: if no container runtime, fall back to `Workdir` with a warning.

### undo ledger

file: `crates/inout-ext-undo/src/lib.rs`

```rust
pub struct UndoLedger {
    snapshots: HashMap<PathBuf, String>,  // path -> pre-edit content
}

impl UndoLedger {
    pub fn snapshot(&mut self, path: &Path, content: String);
    pub fn restore(&self, path: &Path) -> Option<&str>;
    pub fn list(&self) -> Vec<&PathBuf>;
}
```

`/undo <path>` restores. `/undo` (no arg) restores most recent.

### audit trail

covered by v2.1. security-relevant: every `permission_decision` gets an audit record with the tool name, the decision, and the reason (if blocked).
