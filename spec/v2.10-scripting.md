created: 2026-07-19

## scope

the rhai scripting tier: runtime-loaded `.rhai` extensions that register tools, commands, and hooks through the same `ExtensionApi` used by rust crate extensions. this is the user-facing extension surface. first-party rust crates handle performance-critical or security-critical features; rhai scripts handle custom glue, project-specific workflows, and third-party integrations.

### in scope

1. script discovery paths.
2. script lifecycle: load, parse, register, run hooks, unload.
3. the `api` object exposed to scripts.
4. host functions callable from scripts.
5. event representation as rhai maps.
6. tool/command handler signatures.
7. hot-reload during development.
8. permissions: scripts are untrusted by default, all host functions gated.

### not in scope

- implementation of the rhai `Engine` wrapper (the core crate owns that).
- specific first-party extensions (see v2.1–v2.9).

## design rationale

pi has no embedded scripting; everything is ts compiled into the runtime. zap has none either. we want a lean rust core plus the ability for users to extend without recompiling. rhai is the natural choice: rust-native expression language, `no-std` optional but not required, sandboxed by default, easy to call rust functions from.

we do not embed lua, wasm, or deno. those add foreign runtimes and violate "rust-native". rhai is the only interpreter.

## script discovery

scripts are loaded from:

1. `~/.inout/extensions/*.rhai` — user-global scripts.
2. `.inout/extensions/*.rhai` — project-local scripts.
3. paths listed in `config.extension_paths`.

order is significant: user-global first, then project-local, then additional paths. within a directory, files are sorted alphabetically. a script's registered name is its file stem (e.g. `foo.rhai` registers as `foo`). duplicate names across directories are allowed but last-loaded wins on transform hooks.

## script lifecycle

file: `crates/inout-core/src/scripting.rs`

```rust
pub struct ScriptExtension {
    name: String,
    engine: rhai::Engine,
    ast: rhai::AST,
}

impl ScriptExtension {
    pub fn from_file(path: &Path) -> Result<Self>;
}

impl Extension for ScriptExtension {
    fn name(&self) -> &'static str;
    fn version(&self) -> &'static str { "rhai" }
    fn register(&self, api: &mut ExtensionApi);
}
```

load:
1. read file.
2. create isolated `rhai::Engine`.
3. register host functions (read-only subset by default; write/shell/network functions require `INOUT_SCRIPTS_ALLOW_WRITE/SHELL/NETWORK` env flags or config).
4. parse ast. on parse error, log and skip.
5. at registration time, call the script's `register(api)` function.

each script gets its own `Engine`. a panic or exception in one script does not affect others. the `Engine` is `sync` enabled because scripts may be invoked from async hooks.

## the `api` object

within `register(api)`, `api` is a rhai map with these keys:

| key | value | description |
|---|---|---|
| `register_tool` | function(name, desc, schema_json, handler) | register a new tool |
| `unregister_tool` | function(name) | remove a tool from the registry |
| `register_command` | function(name, desc, handler) | register a slash command |
| `unregister_command` | function(name) | remove a command |
| `on` | function(event_type, handler) | subscribe to a typed hook |
| `observe` | function(handler) | observe all events read-only |
| `config` | map | read-only snapshot of config |
| `host` | map | host functions |

### register_tool

```rhai
api.register_tool(
    "my_grep",
    "search files for pattern",
    `{
        "type": "object",
        "required": ["pattern"],
        "properties": {
            "pattern": { "type": "string" },
            "path": { "type": "string" }
        }
    }`,
    fn(args) {
        let pattern = args.pattern;
        let path = args.path;
        let out = api.host.run_command("rg", [pattern, path]);
        return out.stdout;
    }
);
```

handler signature: `fn(args: Map) -> String`. `args` is a map parsed from the llm's tool-use json. return a string (the tool result content). throw an exception to produce an error result.

### register_command

```rhai
api.register_command(
    "todo",
    "list open todos",
    fn(args) {
        let todos = api.host.run_command("git", ["grep", "-n", "TODO"]);
        return todos.stdout;
    }
);
```

handler signature: `fn(args: Array) -> String`. the binary prints the returned string.

### on (hooks)

```rhai
api.on("tool_call", fn(event) {
    if event.name == "bash" {
        // block bash from scripts unless explicitly allowed
        return #{ block: true, reason: "scripts cannot invoke shell" };
    }
    return #{ block: false };
});
```

handler signature depends on event type. return is always a map. see event maps below.

### observe

```rhai
api.observe(fn(event) {
    api.host.log("debug", event.type);
});
```

read-only. return ignored. useful for logging/metrics from scripts.

## host functions

the `api.host` map exposes:

| name | signature | default permission | description |
|---|---|---|---|
| `read_file` | `fn(path: string) -> string` | Read | read file contents |
| `write_file` | `fn(path: string, contents: string)` | Write | write file contents |
| `run_command` | `fn(cmd: string, args: array) -> map` | Shell | run a command, returns `{ stdout, stderr, status }` |
| `http_get` | `fn(url: string) -> string` | Network | simple http get |
| `http_post` | `fn(url: string, body: string) -> string` | Network | simple http post |
| `now_unix_ms` | `fn() -> int` | Read | current unix timestamp ms |
| `log` | `fn(level: string, message: string)` | Read | emit log line |
| `config_get` | `fn(key: string) -> ?` | Read | read config extra key |

permission classes:
- Read: allowed by default.
- Write: requires `INOUT_SCRIPTS_ALLOW_WRITE=1` or `config.scripts.allow_write = true`.
- Shell: requires `INOUT_SCRIPTS_ALLOW_SHELL=1` or `config.scripts.allow_shell = true`.
- Network: requires `INOUT_SCRIPTS_ALLOW_NETWORK=1` or `config.scripts.allow_network = true`.

if a script calls a disallowed host function, the engine throws an error. the error is caught by the tool/command dispatch and returned as an error tool result / printed message.

## event maps

rhai scripts do not see typed rust structs. events are passed as rhai maps. the map always contains `type: string`. additional keys depend on event type.

| event type | map keys |
|---|---|
| `before_agent_start` | `{ type, messages: array, system_prompt: string }` |
| `context` | `{ type, messages: array }` |
| `before_provider_request` | `{ type, stream_options: map }` |
| `before_provider_payload` | `{ type, payload: string }` |
| `before_provider_headers` | `{ type, headers: map }` |
| `after_provider_response` | `{ type, status: int, headers: map }` |
| `tool_call` | `{ type, name: string, id: string, args: map }` |
| `tool_result` | `{ type, name: string, id: string, content: string, is_error: bool }` |
| `message_end` | `{ type, message: map }` |
| `turn_end` | `{ type }` |
| `agent_end` | `{ type }` |
| `session_start` | `{ type }` |
| `session_end` | `{ type }` |
| `model_select` | `{ type, model: string }` |

message map format:

```rhai
# {
#   role: "user" | "assistant" | "tool",
#   content: [
#     { type: "text", text: "..." },
#     { type: "tool_use", id: "...", name: "...", input: {...} },
#     { type: "tool_result", tool_use_id: "...", content: "...", is_error: false }
#   ]
# }
```

for transform events, the handler returns a map with the same keys as the input. the returned map replaces the event for the next handler. for `tool_call`, return `#{ block: true, reason: "..." }` or `#{ block: false }`.

## hot-reload

for development, set `INOUT_SCRIPTS_HOT_RELOAD=1`. then on each new turn, the core re-discovers and re-parses scripts. scripts that have changed are re-registered (tools/commands/hooks are replaced). unchanged scripts are kept to avoid recompilation cost.

in production, scripts are loaded once at startup.

## example script

file: `.inout/extensions/standup.rhai`

```rhai
fn register(api) {
    api.register_command("standup", "show yesterday's commits", fn(args) {
        let since = if args.len > 0 { args[0] } else { "1.day.ago" };
        let out = api.host.run_command("git", ["log", "--oneline", "--since", since]);
        return out.stdout;
    });

    api.observe(fn(event) {
        if event.type == "turn_end" {
            api.host.log("info", "turn completed");
        }
    });
}
```

## acceptance

- [ ] rhai script with `register(api)` function loads and registers a tool.
- [ ] registered rhai tool receives args map and returns a string.
- [ ] rhai tool throwing an exception produces an error tool result.
- [ ] rhai script can register a command and the binary dispatches it.
- [ ] rhai script can subscribe to `tool_call` and block a tool.
- [ ] rhai script can subscribe to `observe` and see event types.
- [ ] disallowed host function throws when env/config flag is not set.
- [ ] hot-reload detects a changed `.rhai` file and re-registers it.
- [ ] parse errors in one script do not prevent other scripts from loading.
- [ ] `cargo clippy --all-targets -- -D warnings` green.
