# twobobs openspec v1.0

project: twobobs
version: 0.1.0
created: 2026-07-18

## scope

this document specifies the minimal rust-native ai agent. v1 includes the conversation loop, five tools, single-agent operation, jsonl history, and a replay-based llm client for tdd. v2+ are explicitly excluded.

### in scope (v1)

1. conversation loop (`agent::run_turn`) with deterministic state machine.
2. five tools: `read`, `write`, `edit`, `grep`, `bash`.
3. filesystem jail: all file tools constrained to `repo_root`.
4. safe bash mode: allowlist, blocklist, no `rm -rf`, no command substitution, no redirection in safe mode.
5. jsonl history persistence and round-trip.
6. replay llm client for offline tests.
7. unit tests for jail and state machine.

### not in scope (v1)

- streaming responses
- subagents, hub coordination, or parallel tool execution
- mcp, browser, debug, lsp, ast edit tools
- skills, managed skills, todos, or learn mechanism
- slash commands
- http llm provider wiring
- spec-driven workflow execution by the agent itself

## components

### state machine

file: `src/state.rs`

states:
- `awaiting_user`: idle, ready for next user message.
- `thinking`: llm call in progress.
- `tool_running`: one or more tools executing.
- `responding`: final text response ready.

events:
- `user_message`: user input received.
- `tool_calls`: assistant requested tool execution.
- `tools_done`: all tool results collected.
- `final_response`: assistant produced text without further tool calls.
- `turn_complete`: response delivered, return to awaiting_user.

legal transitions:

| current | event | next |
|---|---|---|
| awaiting_user | user_message | thinking |
| thinking | tool_calls | tool_running |
| thinking | final_response | responding |
| tool_running | tools_done | thinking |
| responding | turn_complete | awaiting_user |

illegal transitions preserve the current state. this is verified by `tests/state.rs`.

### filesystem jail

file: `src/jail.rs`

rules:
1. absolute paths are rejected.
2. paths containing `..` segments are rejected.
3. resolved path is canonicalised and must start with the canonicalised `repo_root`.
4. symlinks that escape `repo_root` are rejected by rule 3.

errors:
- `jailerror::absolute(path)`
- `jailerror::escapes(path)`
- `jailerror::outside(path)`
- `jailerror::io(err)`

verified by `tests/jail.rs`:
- relative path inside repo resolves.
- absolute path rejected.
- `..` escape rejected.
- symlink escape rejected.
- nested relative path resolves.

### tools

file: `src/tools/mod.rs`

all tools implement the `tool` trait:

```rust
pub trait tool: send + sync {
    fn name(&self) -> &'static str;
    fn schema(&self) -> value;
    async fn run(&self, args: value) -> result<string, toolerror>;
}
```

tool schema is a json object describing the function name, description, and input parameters.

`registry::dispatch` finds the tool by name, runs it, and returns either the tool output or an error string prefixed with `error: `.

#### read

file: `src/tools/read.rs`

input: `{ "path": string, "offset"?: uint, "limit"?: uint }`

behaviour:
1. resolve `path` through jail.
2. read file as utf-8.
3. if `offset` > 1 or `limit` > 0, return only the requested line window.
4. `offset` is 1-based; `limit` of 0 means "to end".

#### write

file: `src/tools/write.rs`

input: `{ "path": string, "content": string }`

behaviour:
1. resolve `path` through jail.
2. create parent directories if missing.
3. write content, replacing existing file.
4. return summary string with byte count and resolved path.

#### edit

file: `src/tools/edit.rs`

input: `{ "path": string, "old_text": string, "new_text": string }`

behaviour:
1. resolve `path` through jail.
2. read file as utf-8.
3. if `old_text` is not present, return error.
4. replace all occurrences of `old_text` with `new_text`.
5. write file back.

#### grep

file: `src/tools/grep.rs`

input: `{ "pattern": string, "path"?: string }`

behaviour:
1. resolve `path` (default `.`) through jail.
2. invoke `rg --line-number --fixed-strings <pattern> <target>`.
3. return stdout. exit code 1 (no matches) returns empty stdout, not error.

#### bash

file: `src/tools/bash.rs`

input: `{ "command": string }`

behaviour:
1. parse `command` with `shell_words` into argv list.
2. reject empty commands.
3. block binaries: `rm`, `dd`, `mkfs`, `parted`, `fastboot`, `shutdown`, `reboot`, `sudo`, `curl`, `wget`, `chmod`, `chown`.
4. reject commands containing `rm -rf`, backticks, or `$(`.
5. in safe mode (`bash.full = false`):
   - binary must be in `bash.allowlist`.
   - reject any token containing `>` (redirection).
6. execute binary with args directly, not through a shell.
7. enforce timeout from config.
8. on non-zero exit, return `command` error with exit code and stderr.

default allowlist: `cargo`, `cat`, `diff`, `echo`, `git`, `ls`, `rg`, `wc`.

### history

file: `src/history.rs`

messages have a role (`user`, `assistant`, `tool`), content, optional tool calls, and optional tool call id.

`history::to_jsonl` serialises the message list as newline-delimited json.
`history::from_jsonl` reconstructs the message list.

`history::to_request` builds an `llmrequest` containing the model name, messages, and tool schemas.

### llm client

file: `src/llm.rs`

trait:
```rust
pub trait llmclient: send + sync {
    async fn complete(&self, req: llmrequest) -> anyhow::result<llmresponse>;
}
```

v1 ships two implementations:
1. `replayllmclient`: returns the next pre-recorded response from a vec. used for offline tests and development.
2. `httpllmclient`: placeholder for anthropic/openai http wiring. intentionally not implemented in v1.

### agent

file: `src/main.rs`

`agent::new(config, llm)` constructs an agent with a tool registry scoped to `config.repo_root`.

`agent::run_turn(user_msg)`:
1. transition to `thinking`.
2. append user message to history.
3. loop:
   - build llm request from history and tool schemas.
   - call llm.
   - if no tool calls, transition to `responding`, append assistant content, return it.
   - otherwise transition to `tool_running`, append assistant content with tool calls, execute each tool, append tool results, return to `thinking`.

## configuration

file: `src/config.rs`

```toml
repo_root = "."
llm_provider = "anthropic"
model = "claude-sonnet-4-5-20250929"
max_turns = 20

[bash]
allowlist = ["cargo", "cat", "diff", "echo", "git", "ls", "rg", "wc"]
full = false
timeout_secs = 30
```

defaults are hard-coded in `config::default()`.
