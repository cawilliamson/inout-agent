# inout openspec v1.0 — core

project: inout
version: 0.3.0-dev
created: 2026-07-19

## scope

the core crate (`inout-core`) is the substrate: the agent loop, the data types, the provider trait, a replay client for tests, the hook bus, the tool/command registries, the extension trait + loader, the rhai runtime, minimal config, and an in-memory session. the core has no tools, no jail, no audit, no persistence, no context management, no skills, no http provider, no observability. all of those are extensions (v2.x).

### in scope (core)

1. data types: `Message`, `ContentBlock`, `Usage`, `AgentEvent`, `ToolError`, `PermissionClass`.
2. `LlmProvider` trait.
3. `ReplayLlmClient` — returns next pre-recorded response. for offline tests of the core.
4. `HookBus` — typed pub/sub. `observe`, `on`, `emit`.
5. `Tool` trait + `ToolRegistry` — dynamic registration, active sets, schema export, dispatch.
6. `Command` trait + `CommandRegistry` — slash commands.
7. `Extension` trait + `ExtensionApi` — the registration surface.
8. extension loader — discovers rust crates (compiled in, feature-gated) and rhai scripts (runtime-loaded).
9. rhai runtime — `ScriptExtension` wraps a parsed rhai ast. discovery, parsing, host function exposure.
10. `Config` — minimal bootstrap: model, provider, repo_root, extension_paths, max_turns.
11. `Session` — in-memory `Vec<Message>` + `State` (the state machine). no persistence.
12. agent loop — `run_turn(user_msg)`. fires hooks at each transition.

### not in scope (core)

- any tool (read/write/edit/grep/bash) — `inout-ext-builtin-tools`.
- jail — `inout-ext-jail`.
- permissions — `inout-ext-permissions`.
- session persistence — `inout-ext-sessions`.
- context management (window, summarizer, edit ledger) — `inout-ext-context`.
- skills — `inout-ext-skills`.
- http provider — `inout-ext-http-provider`.
- observability — `inout-ext-observability`.
- tui/cli rendering — the binary crate or a future tui extension.

### state machine

file: `crates/inout-core/src/state.rs`

states:
- `awaiting_user`: idle, ready for next user message.
- `thinking`: llm call in progress.
- `tool_running`: one or more tools executing.
- `responding`: final text response ready.

events:
- `user_message`: user input received.
- `tool_calls`: assistant requested tool execution.
- `tools_done`: all tool results collected.
- `final_response`: assistant produced text without further tool calls.
- `turn_complete`: response delivered, return to awaiting_user.

legal transitions:

| current | event | next |
|---|---|---|
| awaiting_user | user_message | thinking |
| thinking | tool_calls | tool_running |
| thinking | final_response | responding |
| tool_running | tools_done | thinking |
| responding | turn_complete | awaiting_user |

illegal transitions preserve the current state. verified by `tests/state.rs`.

### hook bus

file: `crates/inout-core/src/hooks.rs`

```rust
pub trait HookEvent: Send + 'static {
    type Result: Default + Send;
    const TYPE: &'static str;
}

pub type HookHandler<E> = Box<dyn Fn(&E, &mut HookCtx) -> E::Result + Send + Sync>;

pub struct HookBus {
    observers: Vec<Box<dyn Fn(&dyn Any) + Send + Sync>>,
    handlers: HashMap<&'static str, Vec<Box<dyn Fn(&dyn Any, &mut HookCtx) -> Any + Send + Sync>>>,
}

impl HookBus {
    pub fn observe<F>(&self, f: F) -> SubscriptionId
        where F: Fn(&dyn Any) + Send + Sync + 'static;
    pub fn on<E: HookEvent, F>(&mut self, f: F) -> SubscriptionId
        where E: HookEvent, F: Fn(&E, &mut HookCtx) -> E::Result + Send + Sync + 'static;
    pub async fn emit<E: HookEvent>(&self, event: E) -> E::Result;
    pub fn clear(&mut self);
}
```

`HookCtx` carries `&mut` to session-relevant state (config, active tools, audit handle if an extension registered one). handlers run in registration order. transform events chain (handler B sees A's output). block events early-exit on `block: true`.

```rust
pub struct ScriptExtension {
    name: String,
    ast: rhai::AST,
    engine: rhai::Engine,

impl ScriptExtension {

}

impl Extension for ScriptExtension {
    fn name(&self) -> &'static str { &self.name }
    fn version(&self) -> &'static str { "script" }
    fn register(&self, api: &mut ExtensionApi) {
        // call the script's `register(api)` function.
        // the api object is a rhai Map with register_tool, register_command, on, observe, config, host.
    }
}
```

the `api` map exposed to scripts:
- `api.register_tool(name, description, schema_json, handler)` — handler is a rhai closure taking a map, returning a string.
- `api.register_command(name, description, handler)` — handler takes an array, returns a string.
- `api.on(event_type, handler)` — handler takes an event map, returns a result map.
- `api.observe(handler)` — handler takes any event, return ignored.
- `api.config` — read-only map of config values.
- `api.host` — host functions: `read_file`, `write_file`, `run_command`, `http_get`, `http_post`, `now_unix_ms`.

full rhai surface is specified in `v2.10-scripting.md`.

### agent loop

file: `crates/inout-core/src/loop.rs`

```rust
pub struct Agent {
    pub config: Arc<Config>,
    pub session: Session,
    pub provider: Box<dyn LlmProvider>,
    pub tools: ToolRegistry,
    pub commands: CommandRegistry,
    pub hooks: HookBus,
}

impl Agent {
    pub fn new(config: Config, provider: Box<dyn LlmProvider>, api: ExtensionApi) -> Self;

    pub async fn run_turn(&mut self, user_msg: String) -> Result<String> {
        // 1. transition to thinking.
        // 2. append user message to session.
        // 3. emit before_agent_start (first turn) — can inject messages / rewrite system prompt.
        // 4. loop:
        //    a. emit context — can transform messages.
        //    b. build llm request from session.messages + tool schemas + system prompt.
        //    c. call provider.
        //    d. emit after_provider_response.
        //    e. if no tool calls: transition to responding, append assistant content, return it.
        //    f. otherwise: transition to tool_running, append assistant content with tool calls.
        //       for each tool call:
        //         - emit tool_call — can block.
        //         - if blocked, append error tool result, continue.
        //         - dispatch tool.
        //         - emit tool_result — can patch result.
        //         - append tool result to session.
        //       transition to thinking, continue loop.
        // 5. emit turn_end.
        // 6. enforce max_turns guard.
    }
}
```

the loop fires hooks at every step. with no extensions registered, the loop still works — it just has no tools (the provider can't call any), no audit, no context management. a bare core + replay client can run a conversation that produces text responses only.

```rust
#[tokio::main]
async fn main() -> anyhow::Result<()> {
    let config = inout_core::Config::load()?;
    let provider = inout_ext_http_provider::AnthropicProvider::from_config(&config);
    let mut api = inout_core::ExtensionApi::new(&config);

    // first-party rust extensions, compiled in.
    inout_ext_builtin_tools::BuiltinToolsExtension::default().register(&mut api);
    inout_ext_jail::JailExtension::default().register(&mut api);
    inout_ext_permissions::PermissionsExtension::default().register(&mut api);
    inout_ext_sessions::SessionsExtension::default().register(&mut api);
    inout_ext_context::ContextExtension::default().register(&mut api);
    inout_ext_skills::SkillsExtension::default().register(&mut api);
    inout_ext_secret_scanner::SecretScannerExtension::default().register(&mut api);
    inout_ext_observability::ObservabilityExtension::default().register(&mut api);
    inout_ext_undo::UndoExtension::default().register(&mut api);

    // feature-gated extensions.
    #[cfg(feature = "ast")]
    inout_ext_ast::AstExtension::default().register(&mut api);
    #[cfg(feature = "mcp")]
    inout_ext_mcp::McpExtension::default().register(&mut api);

    // user rhai scripts.
    inout_core::load_script_extensions(&mut api, &config.extension_paths);

    let agent = inout_core::Agent::new(config, Box::new(provider), api);
    agent.run().await
}
```

## acceptance

- [ ] `inout-core` builds with no extension crates on its dependency list.
- [ ] `inout-core` depends only on `serde`, `serde_json`, `tokio`, `async-trait`, `anyhow`, `thiserror`, `rhai`.
- [ ] state machine transitions verified by `tests/state.rs`.
- [ ] `ReplayLlmClient` returns pre-recorded responses in order.
- [ ] `HookBus::emit` fires handlers in registration order; `observe` handlers see events but cannot mutate.
- [ ] `ToolRegistry::set_active` rejects unknown names and duplicates.
- [ ] `Extension::register` can add a tool, a command, and a hook in one call.
- [ ] `ScriptExtension::from_file` parses a `.rhai` file and registers via `ExtensionApi`.
- [ ] agent loop runs a full turn (user message → llm → tool calls → tool results → final response) with a replay client and a registered test tool.
- [ ] agent loop fires `before_agent_start`, `context`, `after_provider_response`, `tool_call`, `tool_result`, `turn_end` hooks.
- [ ] `cargo clippy --all-targets -- -D warnings` green on `inout-core` alone.
